04

Design Principles 設計思想

01 · 依存関係
Zero Dependency First

Every project defaults to the Go standard library. External packages are added only when they solve a genuine problem — HTML parsing (x/net) and charset conversion (x/text) earned their place. No ORM, no web framework, no reflection magic.

02 · 並行性
Concurrency Safety

All in-memory stores use sync.RWMutex with read/write lock discipline. No global state without protection. The postal code API can handle thousands of concurrent lookups without a single race condition.

03 · 法令準拠
Japanese Law Compliance

The attendance API implements 労働基準法 articles 32, 34, 37, and 39 precisely. The invoice generator supports the 2023 インボイス制度 qualified invoice reform. The holiday API follows 国民の祝日に関する法律 exactly — not a holiday database.

04 · 文字コード
Japanese Encoding Support

Legacy Japanese systems routinely output Shift-JIS and EUC-JP. The price tracker detects charset from Content-Type headers and auto-decodes before HTML parsing — a real-world problem that most Go scraper tutorials ignore.

05 · インターフェース
Interface-Driven Design

The furigana middleware's Annotator interface lets any team swap in MeCab, Kuromoji, or a cloud NLP API without touching the HTTP layer. The price tracker's Scraper can be extended for new platforms by adding a single method.

06 · ドキュメント
Bilingual Documentation

Every README is written in both Japanese and English, formatted for Zenn/Qiita publishing. API documentation includes curl examples, JSON response samples, and Japanese-language field descriptions for team onboarding.
